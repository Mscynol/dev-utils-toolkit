# 安全政策

## 支持的版本

以下版本的 dev-utils-toolkit 目前接收安全更新：

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 支持 |
| < 1.0 | ❌ 不支持 |

## 报告漏洞

如果你发现了安全漏洞，请通过以下方式之一报告：

1. **GitHub Security Advisories**: [提交漏洞报告](https://github.com/Mscynol/dev-utils-toolkit/security/advisories/new)
2. **Email**: 通过 GitHub 个人资料中的联系方式

请不要在公开的 issue 中报告安全漏洞。

## 披露政策

- 报告后 48 小时内确认收到
- 72 小时内提供初步评估
- 漏洞修复后将发布安全公告
- 修复前不会公开披露漏洞细节

## 安全最佳实践

使用本工具库时，建议遵循以下安全实践：

1. **输入验证**: 始终验证用户输入，不要依赖工具库的内部验证
2. **敏感数据处理**: 使用 `mask()` 函数处理敏感数据展示
3. **存储安全**: localStorage/sessionStorage 不适用于存储敏感信息
4. **保持更新**: 及时更新到最新版本
